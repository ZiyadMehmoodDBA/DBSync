# Epic 10A: React Dashboard Foundation — Design Spec

**Goal:** Replace the Vite scaffold placeholder in `src/MSOSync.Frontend/` with a production-ready authenticated shell — working login, auto-refresh, 15 placeholder page routes, `AppLayout` with sidebar + topbar, light/dark theme, and a Vite dev proxy wired to the .NET API.

**Architecture:** React Router v7 nested layout routes. `RootInitializer` bootstraps the session on mount (attempts token refresh). `AuthGuard` outlet enforces authentication for all app routes. `AppLayout` renders the sidebar + topbar shell via `<Outlet />`. Axios intercepts 401 responses with a single-flight refresh lock. The production build embeds in `MSOSync.App/wwwroot` and is served by the .NET API directly — no CORS, no separate server.

**Tech Stack:** React 19, TypeScript, Vite 8, react-router-dom 7, TanStack Query 5, Axios, shadcn/ui, Tailwind CSS 4, Lucide Icons, Zod, React Hook Form, React Flow (`@xyflow/react`), ApexCharts (`react-apexcharts`), ag-grid-community (kept for data tables), Vitest.

## Global Constraints

- TypeScript strict mode; no `any` types
- Tailwind 4 via `@tailwindcss/vite` plugin (not the PostCSS plugin — Tailwind 4 uses the Vite plugin)
- shadcn/ui components installed via CLI (`npx shadcn@latest add <component>`), not bulk-imported
- `react-router-dom` v7 — do NOT install `@tanstack/react-router`
- Access tokens never written to `localStorage` or `sessionStorage`
- localStorage keys: `msosync.refresh_token`, `msosync.user`, `msosync.theme`
- Vite dev server port: 5173; API proxy target: `http://localhost:5000`
- All API calls go through the shared Axios client in `shared/api/client.ts`
- No Zustand — state is React Context + TanStack Query only
- `noUnusedLocals: true`, `noUnusedParameters: true` already in `tsconfig.app.json` — must not introduce dead code
- Theme toggle persists to `msosync.theme` in localStorage; `html.dark` class set before React mounts in `main.tsx` (no FOCT)

## Auth System

### Types (`shared/types/auth.ts`)

```ts
export interface UserProfile {
  username: string;
  roles: string[];
  expiresAt: string; // ISO string from JWT expiresAt
}

export interface AuthState {
  accessToken: string | null;
  user: UserProfile | null;
  isInitializing: boolean;
  login: (username: string, password: string) => Promise<void>;
  logout: () => Promise<void>;
  setTokens: (token: string, refreshToken: string, profile: UserProfile) => void;
  clearTokens: () => void;
}
```

### Token Storage

| What | Where | Key |
|------|-------|-----|
| Access token | React Context memory | — |
| Refresh token | localStorage | `msosync.refresh_token` |
| User profile | localStorage | `msosync.user` (JSON) |
| Theme | localStorage | `msosync.theme` |

### API calls (`shared/api/auth.ts`)

```ts
// POST /api/v1/auth/login → { token, refreshToken, expiresAt }
export async function apiLogin(username: string, password: string): Promise<LoginResponse>

// POST /api/v1/auth/refresh → { token, refreshToken, expiresAt }
export async function apiRefresh(refreshToken: string): Promise<LoginResponse>

// POST /api/v1/auth/logout
export async function apiLogout(refreshToken: string): Promise<void>
```

`LoginResponse = { token: string; refreshToken: string; expiresAt: string }` (camelCase — .NET System.Text.Json defaults).

### Axios client (`shared/api/client.ts`)

```ts
const client = axios.create({ baseURL: '/api/v1' });

// Request interceptor: inject Authorization header from AuthContext
// Response interceptor (401 handler):
let refreshPromise: Promise<void> | null = null; // single-flight lock

client.interceptors.response.use(
  (res) => res,
  async (error) => {
    if (error.response?.status !== 401 || error.config._retry) throw error;
    error.config._retry = true;
    if (!refreshPromise) {
      refreshPromise = doRefresh().finally(() => { refreshPromise = null; });
    }
    await refreshPromise;
    // retry with new token
    return client(error.config);
  }
);
```

`doRefresh()` calls `apiRefresh`, stores new tokens, or calls `logout()` + navigates to `/login` on failure.

The Axios client cannot import from `AuthContext` (circular dependency). Instead, `client.ts` exports a module-level mutable token store that `AuthProvider` populates on mount:

```ts
// shared/api/client.ts — token bridge (no React import)
let _accessToken: string | null = null;
export function setClientToken(token: string | null): void { _accessToken = token; }
export function getClientToken(): string | null { return _accessToken; }
```

`AuthProvider` calls `setClientToken(token)` whenever the access token changes and `setClientToken(null)` on logout. The request interceptor reads `getClientToken()` to inject the `Authorization` header.

### `AuthProvider.tsx`

- Holds `accessToken`, `user`, `isInitializing` in `useState`
- On mount: `isInitializing = true` (blocking `RootInitializer` outlet)
- `login()`: calls `apiLogin`, calls `setClientToken(token)`, stores tokens, sets `isInitializing = false`
- `logout()`: calls `apiLogout` (fire-and-forget), calls `setClientToken(null)`, clears localStorage, clears context state
- `setTokens()`: called after successful refresh (by Axios interceptor); calls `setClientToken(token)`

### `RootInitializer.tsx`

- Renders `<FullscreenLoader />` while `isInitializing === true`
- On mount: reads `msosync.refresh_token` from localStorage
  - If present: calls `apiRefresh` → `AuthContext.setTokens()` → `isInitializing = false`
  - If absent or refresh fails: clears localStorage → `isInitializing = false` → router redirects unauthenticated users to `/login`
- Once `isInitializing === false`: renders `<Outlet />`

### `AuthGuard.tsx`

- Reads `accessToken` from `useAuth()`
- If `null`: `<Navigate to="/login" replace />`
- If present: `<Outlet />`
- Does NOT check `isInitializing` (that is `RootInitializer`'s concern)

### `LoginPage.tsx`

- React Hook Form + Zod schema: `{ username: z.string().min(1), password: z.string().min(1) }`
- On submit: calls `AuthContext.login()`
- On success: `navigate('/dashboard', { replace: true })`
- On failure: display error message below the form
- Uses shadcn/ui `Card`, `Input`, `Button`, `Label`

## Theme System

### Initialization in `main.tsx` (before `createRoot`)

```ts
const theme = localStorage.getItem('msosync.theme') ?? 'light';
document.documentElement.classList.toggle('dark', theme === 'dark');
```

### Toggle (in Topbar)

```ts
function toggleTheme() {
  const next = document.documentElement.classList.contains('dark') ? 'light' : 'dark';
  document.documentElement.classList.toggle('dark', next === 'dark');
  localStorage.setItem('msosync.theme', next);
}
```

Tailwind v4 uses CSS-first configuration. Class-based dark mode is enabled by adding the following to `index.css` (after `@import "tailwindcss"`):

```css
@variant dark (&:is(.dark *));
```

This makes `dark:` utility classes respond to the `.dark` class on any ancestor element (typically `<html>`), which is set by the `main.tsx` initializer and the toggle function above.

## Production Serving (`.NET` changes)

### `src/MSOSync.App/Program.cs` additions

Static file middleware must come **before** authentication so that `index.html` and hashed assets are served without requiring a JWT token. Insert immediately after `app.Build()` (before `app.UseExceptionHandler()`):

```csharp
// Serve React SPA from wwwroot/
app.UseDefaultFiles();
app.UseStaticFiles();
```

Then add the SPA fallback **after** `app.MapControllers()` (must be last):

```csharp
app.MapFallbackToFile("index.html");
```

The resulting middleware order in `Program.cs`:
```
app.UseDefaultFiles();        ← new (before exception handler)
app.UseStaticFiles();         ← new
app.UseExceptionHandler();    ← existing
app.UseRateLimiter();         ← existing
app.UseSecurityHeaders();     ← existing
app.UseAuthentication();      ← existing
app.UseNodeTokenAuth();       ← existing
app.UseAuthorization();       ← existing
app.MapControllers();         ← existing
app.MapFallbackToFile(...);   ← new (last)
```

### MSBuild target in `src/MSOSync.App/MSOSync.App.csproj`

```xml
<Target Name="PublishFrontend" AfterTargets="Publish">
  <Exec Command="npm run build" WorkingDirectory="$(SolutionDir)src/MSOSync.Frontend" />
```

3. Login at `/login` works against the live .NET API

5. Refresh token in localStorage is used to restore the session on page reload
6. 401 responses trigger one automatic refresh then retry; a second 401 logs out
7. Light/dark toggle persists across page reloads with no flash
8. All 3 test suites pass (AuthProvider, AuthGuard, Axios interceptor)
9. `npm run lint` reports 0 errors
